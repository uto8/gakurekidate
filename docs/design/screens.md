## SC-01: ランディングページ

**パス:** `/`  
**目的:** 未認証ユーザーへのアプリ紹介。認証済みの場合は `/discover` へ即リダイレクト。

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヒーロー | ロゴ・キャッチコピー・CTA ボタン |
| フッター | 利用規約・プライバシーポリシーリンク（未定） |

### 配置する情報・操作

- ワードマーク「学歴デート」（ゴールド）
- キャッチコピー（未定）
- 「新規登録」ボタン → `/register`
- 「ログイン」リンク → `/login`

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 正常（未認証） | ヒーローを表示 |
| 認証済み | `/discover` へリダイレクト（画面を表示しない） |

### レスポンシブ方針

- モバイル: フルスクリーン縦積みレイアウト
- タブレット以上: 未定（MVP はモバイルのみ考慮）

## SC-02: ログイン画面

**パス:** `/login`  
**目的:** 既存ユーザーがメールアドレス＋パスワードでログインする。  
**対応要件:** A-2, A-4

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | ロゴ（中央） |
| メインコンテンツ | ログインフォーム |
| フッターリンク | 新規登録画面へのリンク |

### 配置する情報・操作

- メールアドレス入力欄
- パスワード入力欄
- 「ログイン」ボタン（Primary）
- 「アカウントをお持ちでない方は新規登録」リンク → `/register`

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 正常 | フォームを表示 |
| 送信中 | ボタンを `disabled`・`isPending` 表示 |
| エラー | フォーム下部にインラインエラーメッセージ（例: 「メールアドレスまたはパスワードが正しくありません」） |
| 成功 | `/discover` へリダイレクト（学歴未登録の場合は `/onboarding`）|

### レスポンシブ方針

- モバイル: フォームを画面中央に縦積み、横幅 `100%`（横余白 `16px`）
- タブレット以上: フォームを `480px` に制限して中央寄せ

## SC-04: オンボーディング画面

**パス:** `/onboarding`  
**目的:** 新規登録直後に、プロフィール（名前・生年月日・性別・自己紹介・写真）と学歴（大学名・学部・卒業年度）を入力させる。学歴未入力ではメイン機能に進めない。  
**対応要件:** P-1, P-2, P-4

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | ステップインジケーター（ゴールドのプログレスバー） |
| メインコンテンツ | ステップ別フォーム |
| フッター | 「次へ」「完了する」ボタン |

### ステップ構成（案）

| ステップ | 入力項目 |
|---|---|
| Step 1: 基本情報 | 名前・生年月日・性別 |
| Step 2: 自己紹介・写真 | 自己紹介テキスト（任意）・プロフィール写真（任意）|
| Step 3: 学歴 | 大学名・学部名・卒業年度（すべて必須）|

### 配置する情報・操作（Step 1）

- 名前入力欄（必須）
- 生年月日入力（必須・18歳未満は登録不可）
- 性別選択：男性 / 女性（必須）
- 「次へ」ボタン（Primary）

### 配置する情報・操作（Step 2）

- 自己紹介テキストエリア（任意・最大文字数未定）
- プロフィール写真アップロード（任意）
- 「次へ」ボタン / 「スキップ」リンク

### 配置する情報・操作（Step 3）

- 大学名入力欄（必須）
- 学部名入力欄（必須）
- 卒業年度入力欄（必須・数値）
- 「完了する」ボタン（Primary）
- 完了ガイドコピー：「学歴を登録するとすぐに探索が始まります」

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 正常 | 現在のステップのフォームを表示、プログレスバーが進む |
| 送信中（完了ボタン押下後）| ボタン `disabled`・`isPending` 表示 |
| バリデーションエラー | フィールド直下にインラインエラー |
| 写真アップロード中 | プレビューエリアにスピナー表示 |
| 成功 | `/discover` へリダイレクト |
| 学歴未登録でメイン画面アクセス | `(main)/layout.tsx` が検知して `/onboarding` へリダイレクト |

### レスポンシブ方針

- 1 ステップ = 1 スクロール画面に収める（縦スクロール禁止ではないが最小化）
- タブレット以上: フォームを `480px` に制限して中央寄せ

## SC-05: 探索画面

**パス:** `/discover`  
**目的:** 異性ユーザーのプロフィールカード一覧を閲覧し、気になる相手を探す。  
**対応要件:** S-1〜S-4

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | アプリ名（左）・フィルターアイコンボタン（右） |
| メインコンテンツ | プロフィールカードのグリッド（縦スクロール）|
| フッター | ボトムナビゲーション |

### 配置する情報・操作

- プロフィールカード（各カードに以下を表示）
  - プロフィール写真（4:5 縦長・写真なしはイニシャルのグレープレースホルダー）
  - 名前・年齢
  - 大学名・学部名（ゴールド表示）
  - 自己紹介テキスト（2行まで）
- カードタップ → `/profile/[userId]` へ遷移
- フィルターアイコンタップ → OV-01（フィルターパネル）を開く

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 正常 | カードグリッドを表示 |
| ローディング | スケルトンカードをグリッド表示（実際のカードと同レイアウト）|
| 空（該当ユーザーなし） | 「条件に合うユーザーが見つかりませんでした」テキスト + フィルター解除リンク |
| フィルター適用中 | ヘッダーのフィルターアイコンにゴールドのインジケーター（バッジ等）|
| エラー | 「読み込みに失敗しました。再度お試しください」+ リトライボタン |

### レスポンシブ方針

- `< 480px`（モバイル）: 2列グリッド、カード間 `8px`
- `480px〜`（タブレット以上）: 3列グリッド
- カードの最大幅は変えず列数のみ増加

## SC-06: プロフィール詳細画面

**パス:** `/profile/[userId]`  
**目的:** 他のユーザーのプロフィールを詳細表示し、いいねを送る。  
**対応要件:** P-5, L-1, L-3

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー | 戻るボタン（左） |
| メインコンテンツ | 写真・プロフィール情報・学歴 |
| フッター（固定） | いいねボタン（Primary） + ボトムナビゲーション |

### 配置する情報・操作

- プロフィール写真（画面上部フルワイド・4:5）
  - 写真の下端にグラデーションオーバーレイ
  - 名前・年齢を写真上に白テキストで重ねる
- 学歴セクション（ゴールドの左ボーダーライン付き）
  - 大学名・学部名・卒業年度
- 自己紹介テキスト
- 「いいね」ボタン（画面最下部に固定）

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 正常（未いいね） | Primary ボタン「いいね」を表示 |
| 正常（いいね済み）| Secondary ボタン「いいね済み」を表示・タップ不可 |
| いいね送信中 | ボタン `opacity: 0.5`・`pointer-events: none` |
| マッチング成立 | OV-02（マッチングバナー）を表示 |
| ローディング | プロフィール全体をスケルトン表示 |
| エラー | 「プロフィールを読み込めませんでした」+ 戻るボタン |
| 退会済みユーザー | 「退会済みのユーザーです」と表示。いいねボタンを非表示 |

### レスポンシブ方針

- モバイル: 写真はフルワイド。テキストは横余白 `16px`
- タブレット以上: コンテンツを `480px` に制限して中央寄せ、写真も最大 `480px`

## SC-09: チャット画面

**パス:** `/chat/[matchId]`  
**目的:** マッチした相手とリアルタイムでテキストメッセージをやり取りする。  
**対応要件:** C-1〜C-4

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| ヘッダー（固定） | 戻るボタン・相手の名前・大学名 |
| メインコンテンツ | メッセージバブルのスクロールエリア |
| フッター（固定） | テキスト入力欄 + 送信ボタン |

### 配置する情報・操作

- ヘッダー
  - 戻るボタン（← `/matches`）
  - 相手の名前（左）・大学名（小テキスト）
- メッセージバブル
  - 自分のバブル: ゴールド背景・右寄せ
  - 相手のバブル: `bg-elevated` 背景・左寄せ
  - 退会済みユーザーのバブル: グレー背景・「退会済みユーザー」ラベル付き
  - 各バブルの下に送信時刻（`HH:mm`）
- 入力エリア
  - テキスト入力欄（1〜1000文字）
  - 送信ボタン（入力あり時のみアクティブ）
- Realtime で新着メッセージを即時反映・自動スクロール

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 正常 | メッセージ履歴を時系列表示（下が最新）|
| ローディング（初期） | メッセージエリアにスケルトンを表示 |
| 空（メッセージなし）| 「最初のメッセージを送ってみましょう」をメッセージエリア中央に表示 |
| 送信中 | 送信ボタンを `disabled`、楽観的 UI でバブルを先に表示（未定）|
| 送信エラー | バブルをエラー色で表示 + 「再送」ボタン（未定）|
| Realtime 切断 | 「再接続中...」バナーをヘッダー下に表示 |
| 相手が退会済み | 「退会済みのユーザーです。メッセージは送れません」バナーを入力欄上に表示、入力欄を `disabled` |

### レスポンシブ方針

- モバイル: 画面全体を使用。キーボード表示時に入力欄が上にスライド
- タブレット以上: コンテンツを `640px` に制限して中央寄せ

## OV-02: マッチングバナー（オーバーレイ）

**表示元:** SC-06（プロフィール詳細）でいいねを送ったときに相互いいね成立  
**目的:** マッチング成立をユーザーに通知し、チャットへ誘導する。  
**対応要件:** M-2

### 主要な構成要素

| 領域 | 内容 |
|---|---|
| 背景オーバーレイ | `rgba(0,0,0,0.85)` + `backdrop-filter: blur(4px)` |
| 中央カード | マッチング成立の告知 |

### 配置する情報・操作

- ゴールドグラデーションの上部装飾ライン（`3px`）
- 「マッチング成立！」テキスト（Noto Serif JP 22px・ゴールド）
- 自分と相手のアバター（横並び・中央揃え）
- 相手の名前・大学名
- 「メッセージを送る」ボタン（Primary）→ `/chat/[matchId]` へ遷移
- 「あとで」テキストリンク（Ghost）→ バナーを閉じる

### 状態のバリエーション

| 状態 | 挙動 |
|---|---|
| 表示中 | 全画面オーバーレイで前面に表示 |
| 「あとで」タップ | バナーを閉じる（マッチング情報は `/matches` に残る）|
| 「メッセージを送る」タップ | `/chat/[matchId]` へ遷移 |

### レスポンシブ方針

- モバイル・タブレット問わず中央カードを `320px` 固定で表示（未定）
