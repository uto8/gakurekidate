# デザイン原則 — 学歴デート

最終更新: 2026-05-14

---

## ターゲット

**プライマリ：** 恋愛・婚活に真剣な 20 代後半〜30 代の社会人。学歴という共通言語を持つ相手と、誠実に出会いたいと思っている。

**マインドセット：**
- 「誠実な出会い」を求めており、軽いノリのアプリには不信感を持っている
- 学歴を「自慢」ではなく「自己紹介の一部」として自然に開示できる場を探している
- 時間を無駄にしたくない。信頼できる相手とだけ繋がりたい

---

## 印象（ブランドイメージ）

**3つの形容詞：** 洗練・上質・知的

**目指す体験：** 「このアプリを使っている自分を、誰かに見られても恥ずかしくない」

ユーザーが感じるべき第一印象は「品がある」。華美でも地味でもなく、知性的な落ち着き。東カレデートのような大人のトーンを目指しつつ、敷居の高さで離脱させない。

## デザイン原則

### 1. 飾るよりも、伝える

**「情報の豊かさ」よりも「情報の正確さ」を優先する。**

プロフィールカードにできるだけ多くを詰め込むよりも、名前・年齢・学歴という3点を明確に伝えることに集中する。ユーザーが「この人は誰か」をひと目で判断できれば、それで十分。余白は「何もない空間」ではなく「読みやすさのための空間」として積極的に使う。

### 2. 華やかさよりも、誠実さ

**派手なアニメーションや過剰な演出よりも、インタラクションの信頼感を優先する。**

ボタンを押したとき、「ちゃんと反応した」と感じさせることが重要。マッチングの演出も「うるさい祝福」ではなく「静かな喜び」を演出する。動きは「見せる」ものではなく「感じさせる」もの。

### 3. 統一感よりも、学歴の際立ち

**全体の色統一よりも、学歴情報をゴールドで一貫して強調することを優先する。**

ゴールド（`#C9A84C`）は学歴情報専用のシグナルカラー。ユーザーは一覧を流し見するとき、無意識にゴールドの文字を学歴情報として認識するようになる。このシグナルを崩す使い方（アイコンやデコレーションへのゴールド流用）はしない。

### 4. 機能の広さよりも、完成度の高さ

**多くの機能を半端に実装するよりも、少ない機能を磨き込むことを優先する。**

MVP のスコープを守り、未確定の機能に先行してデザインを用意しない。1つの画面が「使っていて気持ちいい」と感じられるまで完成させてから次に進む。中途半端な状態で複数の画面を並行させない。

### 5. 個性よりも、透明性

**ブランドの主張よりも、ユーザーのコンテンツ（写真・プロフィール）が主役であることを優先する。**

背景を黒にするのは「かっこいいから」ではなく、プロフィール写真の色彩を最大限に引き立てるため。UI の存在感を薄くし、人と人とのつながりを前景に出すデザインを選ぶ。

## トーン & ボイス

### 基本方針

**敬語ベースだが、堅苦しくない。** 丁寧語（〜です・〜ます）を使いつつ、ユーザーを「大人として」扱う文体にする。説明しすぎない。

### マイクロコピーの原則

| 原則 | 良い例 | 悪い例 |
|---|---|---|
| 短く、具体的に | 「いいねを送りました」 | 「操作が正常に完了しました」 |
| 感情を押しつけない | 「マッチング成立」 | 「やったー！マッチしました🎉」 |
| ユーザーを主語にしない | 「パスワードが正しくありません」 | 「あなたのパスワードが間違っています」 |
| 次のアクションを示す | 「メッセージを送ってみましょう」 | 「マッチングしました。」（終わり） |
| エラーを責めない | 「もう一度お試しください」 | 「入力に失敗しました」 |

### 画面別のトーン

| 画面 | トーン | 例 |
|---|---|---|
| オンボーディング | 丁寧・案内的 | 「学歴を入力すると、探索が始まります。」 |
| 探索画面 | 静か・観察的 | ラベルや説明を最小限に。写真と名前で語らせる |
| いいね・マッチング | 抑制した喜び | 「マッチング成立」（シンプルに） |
| エラー | 冷静・解決志向 | 「送信できませんでした。もう一度お試しください。」 |
| 退会確認 | 丁寧・中立 | 「退会すると、マッチングやメッセージ履歴は引き継げません。」 |

## やらないこと（避けたいデザイン）

### ビジュアル

- **白背景のライトモード** — コンセプト（夜・大人・洗練）と合わない
- **ゴールド以外のアクセントカラーを追加する** — ブランドのシグナルが薄れる
- **グラデーションの多用** — ロゴとマッチバナーのみ。それ以外に使うと安っぽくなる
- **丸みが強すぎるUI（pill形状ボタンなど）** — 若者向けアプリの印象になる
- **キラキラ・キュートなアイコン** — ポップ感が出てターゲット層に刺さらない
- **フルカラーのイラスト** — 写真主体のアプリと相性が悪い

### インタラクション

- **スワイプUI（Tinder風）** — MVPスコープ外。かつ「軽い出会い」の印象を与える
- **派手な演出アニメーション（confettiなど）** — 「落ち着いたマッチング体験」と矛盾する
- **ポップアップ通知・バッジの多用** — 通知過多はアプリへの不信感につながる
- **モーダルの重ね使い** — 操作の複雑さを感じさせる

### コピー・文言

- **若者言葉・スラング**（「マジ」「ワンチャン」など）
- **絵文字の多用** — 学歴バッジの `🎓` は許容だが、文章中に絵文字を乱用しない
- **過剰な励まし・煽り文句**（「今すぐ出会おう！」「急げ！」など）
- **英語の多用** — サービス名・ラベルは日本語を優先する

### 機能・UX

- **学歴の強制的な格付け・ランキング表示** — 学歴を競争の道具にしない
- **「いいね数」の公開** — プレッシャーと比較を生む
- **自動スクロール・自動再生** — ユーザーの意思決定のペースを尊重する
- **オンボーディングの途中離脱を許さない設計** — 強制感はユーザーを逃がす
