# 要件定義書 — 学歴マッチングアプリ

最終更新: 2026-05-14（矛盾修正: M-1 退会挙動・M-2 年齢フィールド・M-3 学歴必須化）

---

## 1. 概要

### プロダクト名（仮）
学歴マッチング（gakureki-date）

### コンセプト
学歴情報を「信頼性・自己紹介の一部」として活用した、恋愛・婚活向け Web マッチングアプリ。  
学歴は自己申告制とし、マッチングの仕組みは一般的なマッチングアプリ（相互いいね制）と同じ。  
学歴という共通の文脈を持つユーザー同士がより安心して出会える場を提供する。

### ターゲットユーザー
年齢・属性制限なし。恋愛・婚活目的のユーザー全般。

### プラットフォーム
Web アプリのみ（レスポンシブ対応）

### 収益モデル
完全無料（MVP 段階）

---

## 2. ユーザーストーリー

### 新規登録・オンボーディング
- ユーザーとして、メールアドレスとパスワードで登録し、すぐにアプリを使い始めたい
- ユーザーとして、プロフィール（名前・年齢・自己紹介）と学歴（大学名・学部・卒業年度）を登録して、自分を正しく紹介したい
- ユーザーとして、プロフィール写真を任意でアップロードして、相手に顔を見せたい

### 探す・いいね
- ユーザーとして、異性のプロフィール一覧を閲覧して、気になる相手を探したい
- ユーザーとして、年齢・学歴（大学名）でフィルタリングして、条件に合う相手を絞り込みたい
- ユーザーとして、気になる相手に「いいね」を送って興味を示したい
- ユーザーとして、自分にいいねを送ってきた相手の一覧を確認したい

### マッチング
- ユーザーとして、相互にいいねした場合にマッチング成立の通知を受け取りたい
- ユーザーとして、マッチした相手の一覧を確認したい

### チャット
- ユーザーとして、マッチした相手とリアルタイムでメッセージのやり取りをしたい
- ユーザーとして、チャット画面で相手のプロフィール・学歴を確認したい

### プロフィール管理
- ユーザーとして、登録後もプロフィールや学歴情報を自由に編集したい
- ユーザーとして、退会してアカウントと関連データを削除したい

## 3. 機能要件

### 3-1. 認証
| # | 機能 | 詳細 |
|---|---|---|
| A-1 | メール登録 | メールアドレス＋パスワードで新規登録 |
| A-2 | ログイン | メールアドレス＋パスワードでログイン |
| A-3 | ログアウト | セッション破棄 |
| A-4 | 未認証ガード | 未ログイン状態でのアクセスはログイン画面へリダイレクト |

### 3-2. プロフィール
| # | 機能 | 詳細 |
|---|---|---|
| P-1 | プロフィール作成 | 名前・生年月日・性別（男性/女性）・自己紹介・プロフィール写真（任意）。年齢は生年月日から動的計算して表示する |
| P-2 | 学歴入力 | 大学名・学部名・卒業年度（自己申告制）。**必須項目**。オンボーディング完了条件であり、未入力のままでは探索画面に進めない |
| P-3 | プロフィール編集 | 登録後に各項目を編集可能 |
| P-4 | 写真アップロード | Supabase Storage に保存。任意項目 |
| P-5 | プロフィール詳細表示 | 自分・他者のプロフィールを全項目表示 |

### 3-3. ユーザー探索
| # | 機能 | 詳細 |
|---|---|---|
| S-1 | 異性一覧表示 | 自分と異なる性別のユーザーをカード形式で表示 |
| S-2 | 年齢フィルター | 年齢範囲（下限・上限）で絞り込み |
| S-3 | 学歴フィルター | 大学名のキーワード検索で絞り込み |
| S-4 | 自分を除外 | 自分自身は一覧に表示しない |

### 3-4. いいね
| # | 機能 | 詳細 |
|---|---|---|
| L-1 | いいね送信 | 相手プロフィールから「いいね」を送る |
| L-2 | いいね受信一覧 | 自分にいいねを送ってきたユーザーの一覧表示 |
| L-3 | 重複防止 | 同一相手への二重いいねを防止 |

### 3-5. マッチング
| # | 機能 | 詳細 |
|---|---|---|
| M-1 | マッチング成立 | 相互いいねでマッチング成立、DB に記録 |
| M-2 | マッチング通知 | マッチング成立時に画面内通知（バナー等）|
| M-3 | マッチング一覧 | マッチした相手の一覧を表示 |

### 3-6. チャット
| # | 機能 | 詳細 |
|---|---|---|
| C-1 | メッセージ送受信 | マッチした相手とテキストメッセージをやり取り |
| C-2 | リアルタイム更新 | Supabase Realtime で新着メッセージを即時反映 |
| C-3 | メッセージ履歴 | 過去のメッセージを時系列で表示 |
| C-4 | 相手情報表示 | チャット画面ヘッダーに相手の名前・学歴を表示 |

### 3-7. アカウント管理
| # | 機能 | 詳細 |
|---|---|---|
| AC-1 | 退会 | プロフィールをソフトデリート（`deleted_at` をセット）し、ログアウト・探索一覧から除外する。マッチング済みのチャット履歴は相手側の体験を守るため削除しない。退会ユーザーは相手のチャット画面で「退会済みユーザー」と表示される |

## 4. 非機能要件

### パフォーマンス
- ユーザー一覧の初期表示: 2 秒以内
- チャットのメッセージ反映: 1 秒以内（Realtime）

### セキュリティ
- Supabase RLS（Row Level Security）を全テーブルに適用
- 他ユーザーのデータへの不正アクセスを防止
- パスワードは Supabase Auth が管理（平文保存なし）

### 可用性
- Supabase / Vercel の SLA に準拠

### レスポンシブ対応
- スマートフォン（375px〜）からデスクトップまで対応

### アクセシビリティ
- alt テキスト・フォームラベルなど基本的な a11y 対応

---

## 5. スコープ外（MVP では実装しない）

- 学歴の公式認証（学生証・卒業証明書の確認）
- SNS ログイン（Google / Apple 等）
- プッシュ通知・メール通知
- プレミアムプラン・課金機能
- ブロック・通報機能
- 位置情報を使った近くのユーザー表示
- 動画・音声メッセージ
- スワイプ UI（Tinder 風）
- 管理者ダッシュボード
- スマートフォンネイティブアプリ（iOS / Android）
- 同性マッチング

---

## 6. 用語集

| 用語 | 定義 |
|---|---|
| 学歴 | ユーザーが自己申告する最終学歴情報（大学名・学部名・卒業年度）|
| いいね | 他ユーザーへの興味表明アクション |
| マッチング | 二者が互いにいいねを送り合うことで成立する状態 |
| RLS | Row Level Security。Supabase の行レベルアクセス制御機能 |
| Realtime | Supabase のリアルタイムデータ同期機能（WebSocket ベース）|
| MVP | Minimum Viable Product。最小限の機能で動作する初期バージョン |
